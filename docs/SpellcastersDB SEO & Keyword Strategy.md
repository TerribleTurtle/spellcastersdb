### **Executive Summary**

* **Primary Opportunity:** The game combines MOBA mechanics with Deckbuilding 1. This creates a unique "dual-search" behavior where users will search for both *builds* (MOBA terminology) and *decks* (card game terminology). Your site must target both.  
* **Critical Timing:** With Early Access launching in late February 2026, you have a short window to index your database pages.  
* **Strategic Edge:** Unlike generic Wikis (e.g., Fandom 2), a "Database" and "Deck Builder" allows for dynamic content. Your focus should be on *Programmatic SEO*—automatically generating pages for every unit and spell in the JSON data you possess.

### ---

**1\. Keyword Research & Search Intent Strategy**

For a database site, keywords generally fall into three buckets: **Core Brand**, **Database/Entity**, and **Meta/Strategy**.

#### **A. Core Brand & Tool Keywords**

These are high-volume terms users type to find tools. Since the game is free-to-play 3, the player base will likely be large, increasing the volume of these searches.

| Keyword Category | Target Keywords | User Intent |
| :---- | :---- | :---- |
| **Brand Navigation** | Spellcasters Chronicles database, Spellcasters Chronicles wiki, Spellcasters DB | Navigational: Finding a hub for info. |
| **Tool Specific** | Spellcasters Chronicles deck builder, Spellcasters team builder, Spellcasters simulator | Transactional: Users want to *do* something (build/theorycraft). |
| **Game Specific** | Spellcasters Chronicles stats, Spellcasters Chronicles API | Informational: Hardcore users looking for data. |

#### **B. Programmatic Entity Keywords (The "Long Tail")**

Using the JSON data you have, you should target keywords for every specific entity in the game. Users will search for specific cards or heroes to see their stats.

* **Classes:** Fire Elementalist builds, Astral Monk deck, Iron Sorcerer guide, Swamp Witch best spells 4.  
* **Mechanics:** Titan summoning guide, How to capture Source Altars, Spellcasters Chronicles aerial combat tips 6.  
* **Cards/Spells:** You must create a unique URL for every item in your database. Example keywords: \[Spell Name\] stats, \[Unit Name\] cost, \[Unit Name\] counter.

#### **C. Meta & Strategy Keywords (High Traffic)**

Once the game is live, the "Meta" (Most Effective Tactic Available) becomes the highest traffic driver.

* Best Spellcasters Chronicles decks  
* Spellcasters Chronicles tier list 2026  
* Top ranked 3v3 comps  
* Spellcasters Chronicles beginner guide

**2\. Technical SEO & Site Architecture**

To "capture the traffic you need," Google must understand that your site is a structured database, not just a blog.

#### **Programmatic Page Structure**

Your site should ingest the community JSON data to automatically create pages. This scales your site from 5 pages to 500+ pages instantly, casting a wider net for search traffic.

* **URL Structure:**  
  * spellcastersdb.com/decks/ (User-created decks)  
  * spellcastersdb.com/cards/\[card-name\] (Individual card stats)  
  * spellcastersdb.com/classes/\[class-name\] (Class overviews)

#### **Schema Markup**

You should implement **Video Game Schema** and **Item Schema** on your pages. This helps search engines display "rich snippets" (star ratings, stats) directly in search results, which drastically improves Click-Through Rate (CTR).

**Recommendation:**

For a Deck page, use CreativeWork schema to tell Google: "This is a strategy guide created by User X for the game Spellcasters Chronicles."

### ---

**3\. Content Strategy: "The Deck Builder Loop"**

Since you are building a tool, your content strategy should rely on **User Generated Content (UGC)**. This is how sites like *Hearthpwn* or *OP.GG* dominate.

#### **The Flywheel Effect**

1. **Creation:** A user builds a deck using your tool.  
2. **Indexing:** Your site generates a static page for that deck: spellcastersdb.com/decks/fire-elementalist-aggro-v1.  
3. **Sharing:** The user shares this link on the official Discord 7 or Reddit to show off their build.  
4. **Backlinks:** These social shares act as signals to Google that your site is active and relevant.

#### **Priority Content for Launch**

To prepare for the February 26th release, manually create these "pillar" pages:

* **"The Ultimate Founder’s Guide":** Break down the Founder Packs (Explorer, Pioneer, Architect) 8 and recommend which offers the best value.  
* **"Titan Breakdown":** Explain the Titan Gauge mechanics and list all available Titans 4. This is a unique mechanic to this game that will confuse new players, driving search demand.

### ---

**4\. Competitor Landscape & Off-Page Strategy**

Currently, the primary competition is the official site and generic Wikis (e.g., Fandom).

* **Official Site:** Great for lore and news 5, but poor for deep stats, specific numbers, or community builds. You win by being **data-granular**.  
* **Fandom Wiki:** Often slow to load and ad-heavy. You win by having a cleaner, **faster UI** and the interactive Deck Builder tool 2.

#### **Backlink Acquisition Strategy**

* **Community API:** Since you are using the community API, credit the source and reach out to the developers (TerribleTurtle/GitHub). They may link back to you as a "Live Example" of their API in action.  
* **Reddit & Discord:** The community is already active on Discord and Reddit 7. Do not just spam links. Instead, create a "Weekly Meta Snapshot" using data from your site (e.g., "The most popular card this week is X") and post that analysis on Reddit with a link to the data.

### ---

**5\. Recommendations for the "Standard" Package**

You asked what is "standard" for these requests. Beyond the research above, a standard SEO delivery for a database site includes:

1. **Sitemap Submission:** Ensure your dynamic pages (cards/decks) are auto-added to sitemap.xml and submitted to Google Search Console.  
2. **Canonical Tags:** Vital for deck builders. If two users build the exact same deck, you don't want duplicate content penalties. Ensure proper canonicalization logic is in place.  
3. **Page Speed Optimization:** Database sites can get heavy. Ensure your JSON parsing happens efficiently so pages load under 2.5 seconds (Core Web Vitals).
